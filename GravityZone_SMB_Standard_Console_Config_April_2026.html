Before You Start
1
Log into GravityZone Control Center → left sidebar → Policies.
2
Locate the Default policy (read-only). Click the Clone button (top-right menu). Rename the clone: Advatek SMB Standard. Save.
3
Open Advatek SMB Standard for editing. Work through every section below in order. Do not save until all sections are complete — or save section-by-section if the session may time out.
4
After saving: assign to a single test endpoint first. Confirm agent health, definition currency, and Security Center registration before fleet-wide assignment.

4 — Antimalware › On-Access
Antimalware › On-Access
Real-Time File Scanning
On-Access Scanning Enabled Master toggle — must be on
Scan mode Custom Select Custom — then configure individual options below. Do not use Permissive or Aggressive presets
Scan local files All files No extension filtering — scan everything on local drives
Scan network files All files Catches threats coming in via mapped drives and UNC shares
Scan only new and changed files Checked Reduces CPU load on repeat file access; full sweep happens in weekly on-demand scan
Scan boot sectors Checked Catches bootkit and MBR-level threats
Scan process memory Checked Detects in-memory and fileless malware that never touches disk
Scan for keyloggers Checked
Scan for potentially unwanted applications (PUA) Checked Catches adware, toolbars, bundled software — common in SMB environments
Scan inside archives Checked Enable archive scanning — set limits below to manage performance
   Max archive size 100 MB Archives larger than 100 MB are skipped on-access — covered by weekly full scan
   Max archive depth 16 16 nesting levels — sufficient for all real-world archives
Deferred scanning Checked Delays scan of files opened during high-CPU periods — reduces user-visible slowdowns
Action for infected objects Quarantine Quarantine, not Remediate. Preserves file for MSP review. Prevents false-positive data loss on business apps. Set both primary and secondary action to Quarantine
Action for PUA Quarantine Same rationale — review before permanent deletion
5 — Antimalware › On-Execute
Antimalware › On-Execute
Advanced Threat Control, Fileless, Ransomware
Cloud-based Threat Detection Enabled Submits file hashes to Bitdefender cloud for instant threat lookup — zero local performance cost
Advanced Threat Control (ATC)
Advanced Threat Control Enabled Master toggle for ATC — must be on
Detection sensitivity Normal Normal for SMB workloads. Aggressive increases coverage but increases false positives against business apps and custom LOB software. Upgrade to Aggressive only for specific high-risk clients via a separate policy clone
Action for infected applications Quarantine Consistent with On-Access action — quarantine for MSP review
Sensitive Registry Protection Enabled Protects SAM database and critical registry keys from credential dumping tools (Mimikatz, etc.)
   Action Kill process Terminate the offending process immediately — no user prompt
Kernel-API Monitoring Enabled Detects threats that operate at kernel API level — rootkits, injectors, advanced persistence mechanisms
Fileless Attack Protection
Fileless Attack Protection Enabled Blocks malware executing entirely in memory — no file on disk to detect with traditional scanning
Command-line Scanner Enabled Scans command-line arguments at execution time — catches obfuscated PowerShell, WMI, and LOLBin abuse
Antimalware Scan Interface (AMSI) Security Provider Enabled Integrates with Windows AMSI — scans PowerShell, VBScript, JavaScript at runtime before execution. Pairs directly with SMB-INIT-008 PowerShell Script Block Logging
Report analysis results to AMSI Enabled Feeds GZ detection results back into the AMSI pipeline for correlated visibility
Ransomware Mitigation
Ransomware Mitigation Enabled Master toggle — must be on. Requires ATC and On-Access to be active (both are)
EFS Protection Enabled Blocks ransomware from using Windows Encrypted File System as an encryption mechanism
Monitor — Local Enabled Detects ransomware attacks initiated on the local endpoint. Required for all workstations
Monitor — Remote (Network Shares) Enabled Detects ransomware coming from another machine over the network — catches lateral ransomware spread across mapped drives
Recovery Automatic Files restored immediately on detection. No MSP manual intervention required. On-demand option would require MSP to manually trigger recovery from the GZ Reports → Ransomware Activity page within 30 days — automatic is the right choice for SMB
6 — Antimalware › Anti-Tampering
Antimalware › Anti-Tampering
Agent Self-Protection
Anti-Tampering Enabled Master toggle — must be on. Protects the BEST agent from being disabled, killed, or bypassed by malware or an attacker with local access
Vulnerable drivers — Pre-Tampering Deny access Blocks known vulnerable kernel drivers that are commonly used to disable EDR/AV agents (Bring Your Own Vulnerable Driver attacks)
Callback evasion — Post-Tampering Isolate endpoint — Immediately If the agent is actively being disabled, the endpoint is network-isolated instantly. Prevents lateral movement and C2 communication during an active attack

8 — Network Protection
Network Protection modules are included in the core GravityZone license — they are not add-ons. However, they default to Report only, not Block. Every sub-module below must be switched to Block manually.
Network Protection › Web Protection — Antiphishing
Antiphishing
Antiphishing Enabled Master toggle — must be on
Action Block Change from default Report to Block. Phishing pages are stopped before the user can interact with them
Fraud protection Enabled Blocks fraudulent sites (fake banking, fake support, scareware)
Phishing protection Enabled Blocks credential-harvesting pages regardless of URL reputation
Network Protection › Web Protection — Web Traffic Scan
Web Traffic Scan
Web Traffic Scan Enabled Scans all inbound HTTP/HTTPS traffic for malicious payloads
Action for infected websites Block Change from default Report to Block. Connection dropped at the network level before content reaches the browser
TLS handshake interception Enabled Inspects TLS SNI/handshake data to evaluate HTTPS destinations without full SSL inspection. Users see an Access Denied page on blocked sites
Network Protection › Network Attacks — Network Attack Defense
Network Attack Defense (NAD)
Network Attack Defense Enabled Master toggle — must be on
Initial Access techniques Block Covers network-based exploitation attempts (EternalBlue, SMB relay, etc.)
Credential Access techniques Block Covers brute force, pass-the-hash, Kerberoasting over the network
Discovery techniques Block Covers network scanning and enumeration from an attacker foothold
Lateral Movement techniques Block Covers SMB-based lateral movement, WMI exec, PsExec patterns over the network
Crimeware techniques Block Covers C2 communication patterns, ransomware propagation, botnet traffic
Content Control (Web Filtering): Leave unconfigured at SMB Standard tier. Web category blocking is not applied by default — no business justification to restrict browsing categories across all SMB clients. Enable per-client on request with specific category rules. Blocking without client approval creates unnecessary support load.
Excluded Modules — Do Not Configure
Intentional Exclusions
Modules deliberately omitted from SMB Standard
HyperDetect Do Not Enable Billable add-on (Advanced Threat Security). Enabling in the policy without confirming licensing triggers unexpected charges in Syncro billing. Excluded from SMB Standard tier. Upsell to clients who want it
Sandbox Analyzer Do Not Enable Part of the Advanced Threat Security add-on — same billing consideration as HyperDetect
Firewall Do Not Enable Prior deployment caused complete connectivity outage — all RMM communication was blocked, required on-site remediation. Requires a dedicated staged rollout plan before enabling in any environment. Windows Defender Firewall remains active
Device Control Do Not Configure Not applied at SMB Standard tier. Most SMB clients have legitimate USB peripheral use (headsets, scanners, dongles). Enable per-client on explicit request only — reference the Healthcare policy Device Control table as the configuration template
Advanced Anti-Exploit Not Configured System-wide protections (process introspection, privilege escalation, LSASS) are covered by ATC. Custom application protection lists require known process names per client — no standard list applies across all SMB environments. Add per-client when specific high-risk applications are identified

Post-Save Assignment & Verification
After Saving the Policy
Assignment & Verification Steps
1. Assign to test endpoint Required GZ Network → select one non-production endpoint → Assign Policy → Advatek SMB Standard. Wait 5–10 minutes for policy to apply
2. Verify active policy GZ Network → click endpoint → Active policy column must show Advatek SMB Standard
3. Confirm module presence Endpoint Protection tab must show: Antimalware, Advanced Threat Control, Network Protection. If missing modules — run Reconfigure Agent (Mode: Add, Scheduler: Now). Reboot may be required
4. Run GZ Standalone Verification script (Syncro) Confirms EPSecurityService + EPRedLine running, Security Center registered, definitions current. Run manually from Syncro on the test endpoint
5. Fleet assignment GZ Network → select SMB company group → Assign Policy → Advatek SMB Standard. Policy inherits to all endpoints in the group unless an endpoint has an enforced override
