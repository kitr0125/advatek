Overview

This document details the configuration of the Advatek SMB Standard GravityZone policy, built and validated by Advatek Technology Solution in April 2026. The policy is designed for Windows workstations in general SMB business environments — professional services, operations, and office-based workloads — without specific regulatory compliance requirements such as HIPAA.

The policy is cloned from the GravityZone default and explicitly configured across all critical protection layers. The default policy, while adequate for basic detection, leaves HyperDetect off, has no scheduled scan, no Anti-Tampering, no uninstall password, and leaves Network Protection in Report-only mode. This policy corrects all of those gaps while keeping settings calibrated for usability — quarantine over deletion, Normal ATC sensitivity over Aggressive, and no Device Control restrictions that would disrupt standard business workflows.

This document is intended for internal MSP use as a configuration reference. It defines what must be applied in GravityZone Control Center and what modules must be present on endpoints via the Syncro RMM installation package or Reconfigure Agent task.

Configured Sub-Modules
🚫
Antiphishing — Block
Default changed from Report only to Block. Protection against fraud and phishing both enabled. Suspicious pages stopped before the user can interact with them.
🔍
Web Traffic Scan — Block
Scans all inbound HTTP traffic in real time. Infected websites are blocked at the connection level. Action set to Block, not Report only.
🔒
TLS Handshake Interception — On
Inspects TLS SNI/handshake to block HTTPS sites by domain without requiring full SSL inspection. Users see an Access Denied page when a blocked site is reached.
🛡
Network Attack Defense — Block All
All five ATT&CK technique categories set to Block: Initial Access, Credential Access, Discovery, Lateral Movement, Crimeware. Covers SMB exploits, brute force, and lateral movement.

Notes & Design Decisions
HyperDetect Excluded — Billable Add-On
HyperDetect (Advanced Threat Security) is not included in SMB Standard. It is a billable add-on through Syncro/GravityZone. Enabling it in the policy without confirming licensing incurs unexpected charges. This is a deliberate exclusion, not a configuration gap — document it as such when presenting to clients.
→ If a client requests elevated protection, quote Advanced Threat Security as an add-on. Healthcare clients already have it in scope via the Healthcare Hardened policy.
GravityZone Firewall Excluded
The GravityZone Firewall module is intentionally absent. A prior deployment without pre-built RMM port rules caused a complete connectivity outage requiring on-site remediation. Windows Defender Firewall remains active as the workstation-level baseline.
→ Requires a dedicated staged rollout plan before enabling in any environment. Do not enable without pre-testing firewall rules against known RMM communication ports.
Quarantine Over Remediate — Deliberate Choice
On-Access and On-Execute actions are set to Quarantine rather than Remediate. Remediate deletes the detected file and all associated activity (registry keys, child processes). In SMB environments with diverse software, false positives on legitimate business applications are a real risk. Quarantine preserves the file for MSP review and allows recovery without reinstallation.
→ Review GravityZone quarantine weekly. If a pattern of false positives appears for a specific application, add a verified exclusion and release from quarantine.
Device Control Not Applied by Default
USB and external device blocking is not part of the SMB Standard policy. Most SMB clients have legitimate use cases for USB peripherals (printers, scanners, keyboard/mouse, USB headsets). Blanket blocking without client discussion causes operational disruption and support load.
→ Enable Device Control per-client on explicit request. Reference the Healthcare Hardened policy Device Control table for the blocking configuration template. Document the client's device policy in their asset record.
ATC Sensitivity — Normal, Not Aggressive
Advanced Threat Control is set to Normal sensitivity. Aggressive sensitivity increases detection breadth but also increases false positive rate against business applications — particularly custom LOB software, older .NET apps, and scripts used in SMB environments. Healthcare clients run Aggressive because the risk profile justifies it and the app environment is more controlled.
→ If a specific SMB client wants higher sensitivity, it can be overridden with a client-specific policy clone. Document the change and test before fleet deployment.
Safe to Apply to Existing Endpoints
Policy assignment in GravityZone does not reinstall or reconfigure the agent — it only changes behavioral settings. Modules not currently installed on an endpoint remain unaffected until a Reconfigure Agent task is run. Assigning this policy to existing endpoints will apply all antimalware and scan settings immediately and safely.
→ Assign the policy first, then run Reconfigure Agent on endpoints confirmed to be missing required modules. Sequence matters — policy before module install.
