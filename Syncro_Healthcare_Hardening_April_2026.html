Overview

This document details the Syncro RMM automation suite built and validated by Advatek Technology Solution in April 2026 for healthcare workstation deployments. It is the companion reference to the GravityZone Healthcare Hardened Policy document — together they cover both endpoint security (GravityZone) and operating system hardening (Syncro RMM).

The script suite is divided into two categories: 11 one-time initialization scripts that run during provisioning or compliance remediation, and 6 weekly monitoring scripts that run on a recurring schedule to detect drift, verify controls, and auto-create tickets on findings. All scripts are designed to be idempotent where possible — safe to re-run on existing workstations without causing harm.

The suite addresses HIPAA Security Rule technical safeguard requirements under 45 CFR §164.312, covering encryption at rest, audit controls, automatic logoff, access control, integrity monitoring, and transmission security configuration. This document can be presented to clients or auditors as evidence of implemented RMM-layer technical safeguards.

HC-INIT-001 Local Admin Setup Init Creates advadmin / Advatek Support account. Disables built-in Administrator and Guest. Skips creation if account exists, verifies group membership. Access control — unique support account (§164.312(a))
HC-INIT-002 Password & Account Policy Init 90-day max password age, 12-character minimum, 24-password history, complexity enabled, 5-attempt lockout with 30-min auto-unlock. Sets Windows Update auto-install policy registry keys. Password management (§164.308(a)(5)(ii)(D))
HC-INIT-003 BitLocker Enable + Key Escrow Init Enables BitLocker with TPM + Recovery Password protectors. Pre-checks for existing encryption — skips re-encrypt if already protected. Escrows recovery key to Syncro asset fields. Encryption at rest (§164.312(a)(2)(iv))
HC-INIT-004 Windows Security Hardening Init Disables SMBv1, LLMNR, NetBIOS over TCP/IP. Disables TLS 1.0/1.1/SSL 2.0/3.0. Enables TLS 1.2/1.3. Disables RC4/DES/3DES ciphers. Disables WDigest, enforces NTLMv2, restricts anonymous enumeration. Transmission security, protocol hardening (§164.312(e))
HC-INIT-005 HIPAA Audit Policy Init Configures Windows audit subcategories via auditpol: logon/logoff, account management, object access, removable storage, policy change, sensitive privilege use, process creation, system integrity. Expands Security log to 200MB. Audit controls (§164.312(b))
HC-INIT-006 Workstation Lock & Screen Timeout Init Machine-level screensaver: 10-min timeout, password-on-resume, via policy registry key. 15-min inactivity lock. Hides last username on logon screen. Disables Fast User Switching. Limits cached domain logons to 2. Automatic logoff (§164.312(a)(2)(iii))
HC-INIT-007 RDP Security Hardening Init Requires NLA. Sets encryption to High. Enforces 15-min idle timeout and 1-min disconnected session cleanup. Requires user consent for shadowing. Access control, transmission security (§164.312)
HC-INIT-008 Disable Unnecessary Services & Features Init Disables/removes: Remote Registry, Telnet, Xbox services, Windows Media Player Network Sharing, PowerShell v2 (no AMSI/logging support), SMBv1 feature, TFTP client, Windows Insider Service, Telemetry service. Workstation security — attack surface reduction (§164.312)
HC-INIT-009 PowerShell Security Hardening Init Sets execution policy to RemoteSigned. Enables Script Block Logging (Event 4104), Module Logging (all modules), and Transcription to C:\ProgramData\PowerShellLogs. ACL-restricts log directory to SYSTEM and Admins. Audit trail, malicious script detection (§164.312(b))
HC-INIT-010 GravityZone Status Verification Init Verifies EPSecurityService/EPRedLine is running and registered in Windows Security Center. Creates a ticket if agent is not found or not running. Does not modify GravityZone configuration. Antivirus presence verification (§164.312(a)(2)(iv))
HC-INIT-011 Compliance Evidence Report Init Generates an HTML compliance evidence file matching the format requested by HIPAA clients (Yanuvia). Collects BitLocker status, Windows Update policy, AV registration, and screensaver policy. Saves to C:\ProgramData\Advatek\Evidence\{COMPUTERNAME}.html. HIPAA audit documentation — client-requestable evidence
HC-WEEK-001 BitLocker Status Check Weekly Reads current BitLocker state, volume status, and recovery key. Updates Syncro fields. Creates a ticket if status is anything other than Encrypted. Encryption at rest — ongoing monitoring (§164.312(a)(2)(iv))
HC-WEEK-002 Local Admin Account Audit Weekly Compares Administrators group against expected baseline. Flags unexpected accounts, enabled built-in Administrator, and accounts with no password requirement. Creates a ticket on any finding. Access control — privilege audit (§164.312(a)(1))
HC-WEEK-003 Windows Update Compliance Weekly Checks for missing updates via Windows Update COM API. Reports last successful install date. Alerts if patches are older than 30 days. Starts Windows Update service if stopped. Patch management — workstation integrity (§164.312(a)(2)(iv))
HC-WEEK-004 GravityZone Health Check Weekly Verifies GravityZone service is running, install path is present, and agent is registered in Security Center. Attempts service restart if stopped. Creates a ticket if not healthy. Antivirus continuity monitoring (§164.312(a)(2)(iv))
HC-WEEK-005 Failed Login & Lockout Review Weekly Queries Security log for Event 4625 (failed logon) and 4740 (lockout) over 7 days. Groups failures by username. Alerts if any user exceeds 10 failures or if any lockout occurred. Creates a ticket on findings. Audit controls — intrusion detection (§164.312(b))
HC-WEEK-006 System Health & Disk Space Weekly Checks all fixed drives — warning at 15% free, critical at 5%. Detects pending reboots via registry keys. Alerts if uptime exceeds 30 days. Creates Warning or Critical tickets on findings. System availability — operational continuity (§164.312(a))

Notes & Limitations
Password Expiration — 90 Days, Not 30
HC-INIT-002 defaults to 90-day maximum password age. HIPAA does not mandate a specific interval (45 CFR §164.308(a)(5)(ii)(D)) — it requires a procedure for managing password changes. 90 days is the standard accepted by CMS and most HIPAA auditors. 30-day expiration produces weaker passwords in practice.
→ The $MaxPwAgeDays parameter in HC-INIT-002 can be set to 30 if a specific client contract requires it.
Evidence Report — Manual Retrieval
HC-INIT-011 saves the compliance HTML report to C:\ProgramData\Advatek\Evidence\ on the local machine. There is no automated push to OneDrive or a shared location. A technician must retrieve the file via Syncro file manager and place it in the correct client Evidence folder.
→ Future: Build a Syncro script that copies the file to a mapped network share or uses robocopy to sync to a designated path if a UNC is available on the workstation.
Create-Syncro-Ticket — Verify Parameter Names
Multiple weekly scripts call Create-Syncro-Ticket for automated alerting. The exact parameter names (-Subject, -IssueType, -Status, -Body) should be verified against the installed Syncro module version in your environment before deploying at scale. Some versions may use different parameter names or require a CustomerID.
→ Test HC-WEEK-002 on one machine first to confirm ticket creation works as expected before enabling all six weekly scripts.
Existing Workstations — Safe to Re-Run
All 11 INIT scripts are designed to be idempotent. They check before acting (BitLocker pre-checks encryption state, local admin skips if exists, registry keys are simply set/overwritten). Running the full INIT suite on an already-provisioned workstation will verify and enforce all settings without causing harm or resetting configurations.
→ Recommended: Run the full INIT suite on all existing Yanuvia and healthcare client workstations to bring them to baseline compliance.
BitLocker Key Escrow — Recovery
Recovery keys are stored in the Syncro Bitlocker_Key_C asset field and refreshed on every HC-WEEK-001 run. If a key is ever needed during a recovery scenario, pull it from the Syncro asset record. HC-INIT-011 also includes the key in the compliance evidence HTML file generated at provisioning.
→ Confirm key is present in Syncro after provisioning before closing the new workstation ticket.
