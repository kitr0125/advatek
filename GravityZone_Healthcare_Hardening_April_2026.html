Overview

This document details the configuration of the Healthcare Hardened GravityZone policy, built and validated by Advatek Technology Solution in April 2026. The policy is designed for Windows workstations operating in healthcare environments subject to HIPAA Security Rule requirements, including clients running eClinicalWorks and Homecare Homebase via Citrix.

The policy addresses the four core areas of endpoint security relevant to PHI protection: physical exfiltration prevention (Device Control), network-level threat blocking (Network Protection), real-time and behavioral malware detection (Antimalware suite), and exploit protection for clinical application processes (Advanced Anti-Exploit).

This document can be presented to clients or auditors as evidence of implemented technical safeguards under the HIPAA Security Rule (45 CFR §164.312).

Configured Sub-Modules
🚫
Antiphishing — Block
Default action changed from Report only to Block. Protection against fraud and phishing both enabled. Suspicious pages are blocked before the user can interact with them.
🔍
Web Traffic Scan — Block
Scans all inbound HTTP traffic in real time. Infected websites are blocked at the connection level. Action set to Block, not Report only.
🔒
TLS Handshake Interception — On
Inspects TLS SNI/handshake to block HTTPS sites by domain without requiring full SSL inspection. Users see an Access Denied page when a blocked site is accessed.
🛡
Network Attack Defense — Block All
All five ATT&CK technique categories set to Block: Initial Access, Credential Access, Discovery, Lateral Movement, Crimeware. Covers SMB exploits, brute force, and lateral movement attacks.

Note on Homecare Homebase (HCHB): HCHB runs server-side on a Citrix server, not on the local workstation. The three Citrix executables above protect the local Citrix Workspace client session, which is the attack surface on the endpoint. Exploitation of the Citrix client could allow session hijacking and access to HCHB data. Server-side protection (Citrix server endpoint security) is outside the scope of this workstation policy and should be confirmed with the HCHB hosting provider.

Syncro RMM Policy — Required Module Checkboxes
  • Advanced Threat Control — must be checked
  • Device Control (hardware locks, drives, etc) — must be checked
  • Content Control — must be checked
  • Firewall — leave unchecked (connectivity risk — documented incident)
  • Power User — leave unchecked
⚠ Syncro policy changes only affect new installations. Existing endpoints with missing modules must be remediated via the GravityZone Reconfigure Agent task.
