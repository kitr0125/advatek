Overview

This document details the Syncro RMM automation suite built and validated by Advatek Technology Solution in April 2026 for standard small and medium business workstation deployments. It covers baseline provisioning, ongoing compliance monitoring, and on-demand remediation for non-regulated environments.

The suite is divided into three categories: 9 one-time initialization scripts run at provisioning, 5 weekly monitoring scripts scheduled every Monday at 2:00 AM that auto-create tickets on findings, and 3 on-demand remediation scripts run manually from those tickets. All scripts are idempotent where possible — safe to re-run on existing workstations without causing harm.

This suite is designed for standard business clients — construction, trades, retail, professional services, and similar environments — that are not subject to HIPAA, legal privilege requirements, or other regulated compliance frameworks. For regulated environments, refer to the Healthcare Hardening Policy documentation.

Initialization Scripts 9 Scripts — Run at Provisioning
What Gets Configured at Provisioning
👤
SMB-INIT-001 — Local Admin Setup
Creates advadmin / Advatek Support. Disables built-in Administrator and Guest. Verifies Administrators group membership regardless of whether account was just created. Safe to re-run.
⚡
SMB-INIT-002 — Power Plan
High Performance power plan. All sleep/standby timeouts disabled. Detects laptops — flags hibernate disable in notes. GUID fallback if plan not found. Writes to Power Plan Notes asset field.
🔑
SMB-INIT-003 — Password & Account Policy
180-day max age, 10-char min, 10-history, complexity enforced via secedit, 5-attempt lockout, 30-min auto-unlock. Also sets Windows Update auto-install daily at 3AM (AUOptions=4).
🛡️
SMB-INIT-004 — Windows Security Hardening
SMBv1 off, SMB signing required, NTLMv2 enforced, WDigest disabled, LSASS RunAsPPL enabled, TLS 1.0/1.1 and SSL 2.0/3.0 disabled, NetBIOS and LLMNR disabled, AutoRun blocked. Reboot required for LSASS PPL.
🔒
SMB-INIT-005 — Workstation Lock
Machine-level 15-minute inactivity lock via screensaver policy and InactivityTimeoutSecs registry key — cannot be overridden by end users. Hides last username. Disables Fast User Switching.
🖥️
SMB-INIT-006 — RDP Security
NLA enforced, high encryption, 15-min idle timeout, 5-min disconnected cleanup, printer/clipboard redirection disabled, Remote Desktop Users group audited and cleared, shadowing requires user consent.
🔧
SMB-INIT-007 — Services & Features
Remote Registry disabled. WinRM disabled. Secondary Logon disabled. Xbox services off. SMBv1 feature removed. PowerShell v2 removed (no AMSI). Telnet, TFTP, Internet Explorer removed. Print Spooler left enabled.
💻
SMB-INIT-008 — PowerShell Security
Script Block Logging (Event 4104), Module Logging (all modules), Transcription to C:\ProgramData\PowerShellLogs. ACL-restricted log directory. ExecutionPolicy: RemoteSigned. Registry readback verification after applying.
🧹
SMB-INIT-009 — Windows 11 Debloat
Removes Bing apps, Solitaire, Xbox suite, LinkedIn, TikTok, Spotify, Disney, Netflix, Candy Crush, and OEM trials. Preserves all M365 components. Disables Start menu ads, Wi-Fi Sense, sets telemetry to minimum. DryRun supported.

Notes & Limitations
GravityZone Firewall Excluded
The GravityZone Firewall module is intentionally absent from all scripts. A prior deployment without pre-built RMM port rules caused a complete connectivity outage requiring on-site remediation. Windows Defender Firewall remains active as the workstation-level baseline.
→ Requires a dedicated staged rollout plan before enabling in any environment.
GravityZone Deployment Sequencing
The GravityZone Standalone verification script is intentionally excluded from the INIT sequence. GravityZone is deployed via the Syncro RMM agent and may not be installed by the time INIT scripts complete. Running verification against an absent agent generates false negative tickets.
→ Run the Standalone script manually after confirming GravityZone is installed and the Syncro agent has checked in.
Print Spooler Left Enabled
SMB-INIT-007 leaves the Print Spooler service enabled by default. PrintNightmare (CVE-2021-34527) is a real attack surface but disabling Spooler breaks all local and network printing — not acceptable as a default for most SMB clients.
→ Disable manually on endpoints confirmed to have no printing requirements. Document the exception in the asset record.
Nightly Power-Off Pattern
SMB clients who power off workstations nightly will have inconsistent weekly script coverage. Machines that are off at 2AM Monday will miss that week's compliance checks. SMB-WEEK-005 surfaces this via the uptime floor check and flags it in the ticket.
→ Discuss leaving managed workstations powered on overnight with affected clients. Document client preference if they decline.
BitLocker Not Enforced
Full disk encryption via BitLocker is not enforced at the SMB Standard tier. BitLocker enforcement and weekly key escrow verification is reserved for regulated environments (Healthcare, Legal, Finance). SMB clients may optionally have BitLocker enabled on a per-device basis.
→ If a client requests BitLocker, enable manually and apply the HC-INIT-003 and HC-WEEK-001 scripts selectively.
Safe to Re-Run on Existing Workstations
All 9 INIT scripts are idempotent. They check before acting — accounts are skipped if they already exist, registry keys are simply overwritten, and services check state before attempting changes. Running the full INIT suite on an already-provisioned workstation will verify and enforce all settings without causing harm.
→ Recommended: Run the full INIT suite on existing client workstations being brought under the SMB Standard policy for the first time.
